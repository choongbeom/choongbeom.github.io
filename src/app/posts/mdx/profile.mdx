# 김충범 | Senior Software Engineer

**Windows 기반 장비·SDK와 금융·공공 솔루션을 장기간 개발해 온 소프트웨어 엔지니어입니다.**

2009년부터 약 **17년간** 신분증·문서 스캐너, 본인확인기, 키오스크, ATM 등 장비와 업무시스템을 연결하는 소프트웨어를 개발해 왔습니다.

C++, C#, MFC, WinForms 기반 Windows 응용프로그램과 장비 SDK 개발을 장기 실무 기반으로 보유하고 있으며, 인터페이스 설계, 설치·배포, 고객 환경 적용, 장애 분석과 유지보수까지 제품 생명주기 전반을 경험했습니다.

현재는 개발팀장으로서 팀 운영, 업무 배분, 일정·성과관리, 예산 산정과 채용 참여, 제품의 기술 방향과 핵심 의사결정을 담당하고 있습니다.

2025년에는 금융·공공기관의 대면창구와 업무환경을 중심으로 여러 신규 제품 및 사업 아이디어를 검토하고, 일부 과제는 시스템 구조 설계와 데모 애플리케이션 개발까지 수행했습니다.

2026년부터는 기존 장비 소프트웨어 경험을 바탕으로 Windows Scanner Agent, Web, API, Worker, Database, Queue, Object Storage와 상용 Document AI를 연결한 문서처리 제품을 설계하고 구현하고 있습니다.

특정 기술 하나만을 강조하기보다, 장비·소프트웨어·고객 업무·운영 조건을 함께 이해하고 실제 제품의 형태로 연결하는 데 강점을 가지고 있습니다.

### 신규 제품·사업 기술 검토 및 PoC | 2025

2025년에는 금융·공공기관의 대면창구와 업무환경을 중심으로 신규 제품 및 사업 아이디어의 기술·시장 타당성을 검토했습니다.

과제별 목적과 우선순위에 따라 시장·법제 조사, 제품 개념 정의, 시스템 아키텍처, 화면·데이터·API 설계, 하드웨어 검토와 데모 애플리케이션 개발까지 서로 다른 단계로 진행했습니다.

**대면창구 통합 관제 시스템**

금융·공공기관 창구 PC와 주변장비 상태를 중앙에서 관리하기 위한 시스템을 기획했습니다.

- 창구 PC에 설치되는 Windows Agent 구조 설계
- 장비와 단말 상태 수집·전달 방식 정의
- 중앙 관제 Web 서비스와 API 구조 설계
- Agent의 일부 Web 연동 기능 개발
- 관제 서비스 기본 화면 구현

전사 우선순위 조정으로 본격적인 제품 개발은 보류되었습니다.

**음성 자막 디스플레이 시스템**

금융·공공기관 대면창구에서 상담자의 음성을 실시간 문자로 변환해 디스플레이에 표시하는 제품을 검토했습니다.

- 국내외 대면창구 적용 사례 조사
- 투명 및 탁상형 양면 디스플레이 구매와 하드웨어 분석
- 온디바이스 방식과 외부 연동 방식의 음성인식 구조 비교
- 음성인식 결과를 표시하는 데모 애플리케이션 개발
- 영업본부 대상 실제 시연

기술 검증과 시연까지 수행했으며, 시장 요구와 사업화 우선순위를 고려해 후속 제품화는 보류되었습니다.

**교육청 교권 보호용 음성 분석 시스템**

교육청 민원 창구와 통화 과정에서 발생할 수 있는 폭언·위협 발언을 인지하고, 교직원 보호와 분쟁 대응을 지원하는 시스템을 검토했습니다.

- 교육청 민원 처리 흐름과 활용 시나리오 조사
- 음성 수집, 저장과 분석 지점 정의
- 기존 민원 시스템과 연계 가능한 구조 검토
- 초기 제품 개념과 시스템 구성 방향 설계

시장 성숙도와 도입환경을 고려해 사업 추진 방향이 조정되었습니다.

**건설기초안전교육 이수 확인 시스템**

건설 현장에서 사용되는 기초안전교육 이수증의 위·변조와 확인 절차 개선 가능성을 검토했습니다.

- 관련 법령과 제도 조사
- 실제 건설 현장의 확인·관리 절차 분석
- 이수증 위조 사례와 운영 한계 조사
- 전산 확인 시스템의 기대효과와 운영구조 검토

시장 규모와 제도환경을 고려한 사업성 검토 후 추진이 보류되었습니다.

**디지털 자료관 AI 고도화**

공공기관의 기존 디지털 자료관에 AI 기술을 적용하여 검색과 활용성을 높이는 방안을 검토했습니다.

- 국내 디지털 자료관의 운영 현황과 문제점 조사
- AI 검색, 요약과 번역 방식 비교
- 기존 시스템을 유지하면서 확장할 수 있는 구조 설계
- 공공 조달 구조와 사업 확장 가능성 검토

중·장기 과제로 분류되어 단기 추진 대상에서는 제외되었습니다.

**전자금융업자 신분증 진위확인망 확대 대응**

전자금융업자에게 신분증 진위확인 서비스가 확대될 가능성에 대비하여 당사가 기술 파트너로 참여할 수 있는 방안을 검토했습니다.

- 신분증 진위확인 서비스 구조와 전자금융업권 조사
- 기존 신분증 스캔·정보 추출 기술의 활용 가능성 분석
- 연동 방식별 대응 시나리오 정리
- 단계별 사업 및 기술 대응 전략 작성

관련 정책과 제도가 확정되기 전 단계로 사업 추진은 보류되었습니다.

**주변기기 컨트롤러**

금융·공공기관 대면창구에서 사용하는 스캐너, 프린터, 카메라 등 USB 주변기기의 연결과 상태를 관리하는 제품을 기획했습니다.

- 창구 주변기기의 연결·운영 문제 분석
- 주변기기 상태관리와 장애 대응 시나리오 정의

- 고객 및 영업 활용을 위한 컨셉 브로셔 제작

**통합 고객 전략 관리 시스템**

- 고객 정보, 영업이력과 전략관리 기능 정의
- 화면 구성과 사용자 흐름 설계

이후 전사 CRM 도입이 결정되면서 자체 시스템 개발 검토는 종료되었습니다.

**ERP 데이터 기반 파트너몰 검토**

최근 5년간 ERP 주문·매출 데이터를 분석하여 파트너 전용 온라인 판매채널의 도입 가능성을 검토했습니다.

- 고객·상품·주문·매출 데이터 분석
- 파트너몰 운영 구조와 수익모델 검토
